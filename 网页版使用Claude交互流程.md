**① 客户端（浏览器）**

你在输入框打完字、点击发送后，浏览器前端（claude.ai 是一个 React 应用）先在本地更新 UI 状态，同时检查你的身份凭证——Session Cookie 或 JWT Token 是否有效。如果未登录或 Token 过期，请求会被拦截并跳转到登录页，根本不会发到服务器。

---

**② 网络层**

请求通过 HTTPS/TLS 加密后发出，Anthropic 在全球部署了 CDN 节点，你的请求会被路由到地理上最近的入口，再转发给后端集群。进入后端之前还有一层速率限制与安全过滤：检测请求频率是否超限、是否携带异常 payload，防止滥用和 DDoS 攻击。

---

**③ 服务端处理**

API 服务器收到请求后做两件关键事：一是拼装完整上下文，把本次对话的历史消息全部合并进来（Claude 没有持久记忆，每次都需要把完整历史带上）；二是注入 System Prompt，也就是你看不见的指令层，包含 Anthropic 的安全规则、产品人格设定等。之后，所有文本被 Tokenizer 切分成"词元"（token），大概每 3-4 个汉字或每个英文单词对应 1-2 个 token，这是模型实际处理的最小单位。

---

**④ 模型推理（最核心的环节）**

Token 序列进入 Claude 模型——一个数百亿参数的 Transformer 神经网络。模型通过注意力机制（Attention）理解每个 token 和其他 token 之间的语义关系，然后逐个预测下一个最合理的 token 是什么。输出时还有采样策略（Temperature、Top-P）控制回答的创意程度与确定性，最后经过输出安全审核层，过滤潜在有害内容。

---

**⑤ 流式回传**

Claude 不会等整个回答生成完才发给你，而是通过 SSE（Server-Sent Events）协议一边生成一边推送，所以你会看到文字像"打字"一样逐字出现。浏览器接收到 token 流后实时渲染 Markdown：代码块、公式、表格都在这一步格式化呈现。

---

**几个常见的疑问**

从你点击发送到看到第一个字，通常需要 0.5–3 秒，这段时间叫"首字延迟"（TTFT），主要消耗在模型开始推理的准备阶段。之后文字出现的速度取决于模型推理速度（通常每秒几十个 token）以及当前服务器负载。上下文越长（对话历史越多），推理耗时越久，因为 Attention 的计算量和上下文长度的平方成正比。
