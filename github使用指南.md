## GitHub 入门介绍

GitHub 是全球最大的**代码托管平台**，简单来说就是一个"云端硬盘"，但专门为代码和文件设计，并加入了强大的版本管理功能。

---

### 核心概念

**Repository（仓库 / Repo）**
就像一个项目文件夹，你的所有学习资料都存放在里面。每个仓库可以设为公开（Public）或私有（Private）。

**Commit（提交）**
每次保存修改就叫一次 Commit。GitHub 会记录每一次改动，你可以随时回到任意历史版本——再也不怕误删文件了。

**Branch（分支）**
可以理解为"草稿"，在不影响主文件的情况下做实验或修改，满意了再合并进去。新手暂时不需要深入了解。

---

### 作为新手存储学习资料，你需要掌握的操作

| 操作 | 说明 |
|------|------|
| 创建仓库 | 点击右上角 `+` → New repository |
| 上传文件 | 进入仓库 → Add file → Upload files |
| 编辑文件 | 直接在网页上点击文件 → 铅笔图标编辑 |
| 查看历史 | 点击 `Commits` 查看所有修改记录 |

---

### 几个实用小建议

1. **README.md** —— 每个仓库建议创建一个 README 文件，用来写这个仓库是做什么的，方便自己日后查找。

2. **善用文件夹分类** —— 上传时可以建立层级结构，比如 `Python/笔记.md`、`数学/公式.pdf`，让资料更整洁。

3. **私有仓库** —— 个人学习资料建议设为 **Private**，只有自己能看到，免费账号也可以无限创建私有仓库。

4. **Markdown 格式** —— GitHub 原生支持 `.md` 文件渲染，用来做笔记非常美观，推荐学习基础 Markdown 语法。
